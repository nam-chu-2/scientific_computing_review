# attempts/

Your scratch files for `script`-mode questions land here as `<id>.py` when the
exam presents them. Edit the function body, then in the exam type:

- `check` — grade it against the (hidden) test suite
- `run` — execute it on the sample input without grading
- `reset` — restore the original starter code

These files are **yours**. They're git-ignored (see `../.gitignore`) so your
work-in-progress doesn't clutter the repo. Delete them anytime; the exam
re-creates the starter when it next presents that question.
